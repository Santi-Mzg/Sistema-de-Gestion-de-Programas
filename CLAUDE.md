## Backend (Spring Boot)

**Directory:** `Silabus-UNS/Backend`

```bash
# Development
./mvnw spring-boot:run

# Build (skip tests)
./mvnw clean package -DskipTests

# Build (with tests)
./mvnw clean package

# Run tests only
./mvnw test

# Run the built JAR
java -jar target/Sistema_de_Gestion_de_Programas-0.9.0-SNAPSHOT.jar
```

Swagger UI: `http://localhost:8080/swagger-ui.html`

**Package layout** (`src/main/java/com/santimaszong/Sistema_de_Gestion_de_Programas/`):

| Package | Role |
|---------|------|
| `controller/` | REST controllers (Auth, Area, Carrera, Departamento, Materia, Programa, User) |
| `domain/entities/` | 13 JPA entities |
| `domain/dto/` | Request/Response DTOs (area, auth, carrera, departamento, materia, programa, user) |
| `domain/enums/` | `EstadoPrograma`, `AccionPrograma`, `Rol`, `TokenType` |
| `repositories/` | Spring Data JPA repositories |
| `services/impl/` | Business logic implementations (one per aggregate) |
| `services/auth/` | Auth service |
| `services/email/` | Async email via Resend API |
| `services/gemini/` | Google Gemini AI integration (WebFlux) |
| `services/pdf/` | PDF generation (Thymeleaf → Flying Saucer → OpenPDF) |
| `mappers/` | MapStruct mappers + hand-written extensions |
| `security/` | JWT filter, password generation |
| `util/exception/` | Global exception handler |
| `util/seeder/` | Initial data seeder |

DB migrations: `src/main/resources/db/migration/` (Flyway).
Thymeleaf templates for PDF: `src/main/resources/templates/`.

**Environment variables:**

| Variable | Purpose |
|----------|---------|
| `SPRING_DATASOURCE_URL` | PostgreSQL JDBC URL |
| `SPRING_DATASOURCE_USERNAME` | DB username |
| `SPRING_DATASOURCE_PASSWORD` | DB password |
| `JWT_SECRET` | JWT signing key |
| `SERVER_PORT` | HTTP port (default 8080) |

## Frontend (Next.js)

**Directory:** `Silabus-UNS/Frontend`

```bash
npm install           # First time setup
npm run dev           # Dev server → http://localhost:3000
npm run build         # Production build
npm start             # Serve production build
npm run generate:api  # Regenerate API client from OpenAPI spec (run after backend changes)
npm run lint          # ESLint
```

**Key conventions:**
- API client (types + React Query hooks) lives in `app/api/generated/` — **generated by Orval** from `/v3/api-docs`. Never edit generated files manually; run `npm run generate:api` after backend changes.
- Next.js config rewrites `/api-proxy/*` to the backend via `NEXT_PUBLIC_API_URL`.
- Forms use **React Hook Form + Zod**.
- Data fetching uses **TanStack Query v5**.

**Environment:**

| Variable | Purpose |
|----------|---------|
| `NEXT_PUBLIC_API_URL` | Backend base URL (default `http://localhost:8080`) |

## Core Domain Model

- **ProgramaEntity** – central entity (syllabus). `EstadoPrograma`: BORRADOR → ENVIADO → APROBADO/RECHAZADO.
- **ProgramaDraftEntity** – draft before formal submission.
- **DecisionComisionEntity** – commission approval/rejection with justification.
- **EstadoHistoricoEntity** – state change audit trail.
- **MateriaEntity** – subject/course (belongs to Departamento + Area).
- **CarreraEntity** + **CarreraPlanEntity** – degree program and study plan.
- **ProgramaCarreraEntity** – N:M between programs and careers.
- **DepartamentoEntity** / **AreaEntity** – organizational hierarchy.
- **UserEntity** – user with `Rol`: Admin, Docente, Comisión.
- **UsuarioDepartamentoEntity** – N:M between users and departments.
- **PasswordTokenEntity** – password reset token.
